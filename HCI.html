    • 1. Introduction - About ASLearner

    ASLearner is a mobile app designed to help the people who cannot speak and the people that desire to learn the sign language; the app contains 3 modules - one for live translation and 2 centered on guided learning - which facilitate the learning of ASL in a different way.

    2. Overall Description

    2.1 Product Perspective

    ASLearner is a new, self-contained product, that is based on a previous project - Sign Language 101 - intended to give a more complex and easier method in learning ASL.

    2.2 Product Functions

    The functions offered by the product are the live translation of ASL signs - Live Translator -, the display and the possibility of interacting with the translated text for the users not knowing the sign language and 2 modules intended for the users who want to learn this language; the first module - Signing Improver - offers the user the possibility of signing certain given ASL characters; in case the user incorrectly signs the character, he is being offered a hint (a displayed pose for the hand) in order to sign the character correctly while the second module - Learn the Basics - is more oriented for the beginner learner. This last module puts the user in a learning position: being given an ASL character and the pose for the hand, the user should make the pose accordingly in order to advance to the next characters. The user is also free to make hand gestures in order to explore the ASL.

    3. Design Specifications

    3.1 Interaction Design Patterns

    The main interaction design patterns used in this project are:

    • One Window Drilldown: Being a mobile app, the window size is reduced; this interaction design pattern is used in conjuction with Breadcrumbs in order to facilitate the easy interaction and the application flow for the user
    • Breadcrumbs: Every window from the second level of the application flow displays a path explored by the user until that window; in this manner, the user can touch the path in order to get a step back and it is also favours recognition and recall by respecting the 3-Click Rule (the longest length of the exploration path from the first window to the last windows is 3 touches).
    • Multi-Level Help: Along with the about section of the app explaining the application modules, the learning modules windows will display certain messages in order to help the client to use the app.
    • Hub and Spoke: The 3 modules are independent one from another; they can be accesed independently by the client after accessing the "Start" menu.
    • Pyramid: The interaction design is structured like a pyramid; at the end points of the "Start" menu of the interaction graph lies one of the 3 modules available for the user, facilitating the hierarchical and sequencial exploration. It is used in conjuction with One Window Drilldown for a better display of the data.
